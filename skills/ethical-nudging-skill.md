# Ethical Nudging Skill

Purpose
- Provide clear ethical guardrails and review patterns for behavioral interventions to prevent dark patterns and ensure user benefit.

Principles
- Respect autonomy: avoid coercion and ensure obvious opt-outs.
- Transparency: make intentions clear when an intervention alters choices.
- Beneficence: interventions should demonstrably improve user outcomes or experiences.
- Proportionality: benefit should outweigh any small frictions introduced.

Red flags (stop & review)
- Hidden defaults that materially disadvantage users
- Deceptive copy or fake scarcity
- Opt-outs that are intentionally difficult or confusing
- Exploitative pricing anchors targeted at vulnerable groups

Review checklist (before running any pilot)
- Does the experiment require explicit consent? If yes, collect it.
- Is the expected benefit for users documented and measurable?
- Are there any vulnerable user groups affected? Apply extra caution.
- Is there a simple, immediate opt-out or undo path?
- Is the intervention reversible without data loss?

Consent language examples
- Optional publish: "I consent to having this summary shown to Leadership for the purpose of review."
- Data sharing: "Uploading this file will store it in Sweden Central and will be visible to Leadership as aggregate unless I opt-in."

Governance
- Log ethical review decision in the proposal (who reviewed, date, notes)
- For borderline cases, require BA + Security + Legal sign-off before rollout
